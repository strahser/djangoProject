# Модуль email_ui

Веб-интерфейс для работы с электронной почтой в рамках Django-проекта.  
Заменяет/расширяет стандартную админку `Emails` современным UI с ночным режимом,
TinyMCE-редактором, массовыми операциями, контактами, тегами, правилами и пр.

## Возможности

### Просмотр писем
- **Список писем** — таблица с сортировкой по отправителю/теме/дате,
  пагинация с бесконечной прокруткой, отображение важности и вложений
- **Фильтры** — по проекту, подрядчику, типу здания, категории,
  типу информации, отправителю, поиск по тексту, дате, наличию вложений
- **Детальная страница** — полный просмотр письма с метаданными,
  тегами, задачами, вложениями, телом письма
- **Сохранённые фильтры** — возможность сохранять и быстро применять
  наборы фильтров

### Отправка писем
- **Новое письмо** — полноэкранный редактор с TinyMCE (WYSIWYG)
- **Ответ / Ответить всем / Переслать** — предзаполнение полей
  и тела письма с цитированием
- **Вложения** — загрузка файлов с удалением ошибочно прикреплённых,
  автоприкрепление вложений оригинала при пересылке
- **Черновики** — автосохранение с подтверждением при закрытии
- **Выбор аккаунта** — SMTP (Yandex) или Outlook

### Управление
- **Массовые операции** — отметить прочитанным/непрочитанным,
  важным/неважным, переместить в папку, удалить в корзину,
  выбрать все в фильтре
- **Папки** — Входящие, Отправленные, Черновики, Архив, Корзина
- **Теги** — назначение/снятие, цветные метки
- **Важность** — автоматическая (при заполнении метаданных)
  и ручная установка звёздочкой
- **Задачи** — привязка существующих задач из ProjectTDL,
  создание новых

### Контакты
- CRUD контактов с несколькими email-адресами
- Быстрый выбор из выпадающего списка при составлении письма
- Поиск контактов в модальном окне

### Правила
- Автоматические действия при получении писем (перемещение,
  назначение тегов, установка важности)

### Шаблоны
- Сохранённые шаблоны писем для быстрой отправки

### Интерфейс
- **Ночной режим** — сохраняется в localStorage
- **Размер шрифта** таблицы — регулируется слайдером
- **Адаптивная боковая панель** с счётчиками непрочитанных
- **Полноэкранное модальное окно** для compose/reply/forward
- **Копия письма в структуру проекта** — копирует файлы в
  `E_MAIL_DIRECTORY/Проект/Подрядчик/Тип/Год/...` и путь в буфер обмена

## Технический стек

- **Backend**: Django 5, Python 3.11
- **База данных**: SQLite (`db.sqlite3` + `personal_db.sqlite3`)
- **Frontend**: Bootstrap 5, HTMX, TinyMCE 6
- **Почта**: IMAP (Yandex), SMTP (Yandex SSL:465), Outlook COM
- **Файлы писем**: HTML на диске, `Email.link` → директория

## Установка и запуск

1. Активировать виртуальное окружение
2. `pip install -r requirements.txt`
3. Настроить `.env`:
   - `YA_USER`, `YA_PASSWORD` — учётные данные Yandex IMAP
   - `E_MAIL_DIRECTORY` — путь к директории переписки
4. `python manage.py migrate`
5. `python manage.py runserver`

## Тестирование

```bash
python manage.py test email_ui.tests --parallel 1
```

## Основные URL

| URL | Описание |
|-----|----------|
| `/email-ui/inbox/{folder}/` | Список писем в папке |
| `/email-ui/email/{pk}/` | Детальная страница письма |
| `/email-ui/compose/` | Новое письмо (модальное окно) |
| `/email-ui/reply/{pk}/{type}/` | Ответ/переслать (модальное окно) |
| `/email-ui/send/` | Отправка нового письма (POST) |
| `/email-ui/reply/{pk}/send/` | Отправка ответа/пересылки (POST) |
| `/email-ui/save-draft/` | Сохранение черновика (POST) |
| `/email-ui/bulk-action/` | Массовые операции (POST) |
| `/email-ui/contacts/` | Список контактов |
| `/email-ui/tags/` | Список тегов |
| `/email-ui/rules/` | Список правил |
| `/email-ui/unread-count/` | Счётчики для боковой панели (JSON) |
